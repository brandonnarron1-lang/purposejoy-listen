# PurposeJoy — Shorts Pipeline

End-to-end workflow for creating and uploading YouTube Shorts from PurposeJoy tracks.

## Prerequisites

- Node.js ≥ 18
- ffmpeg on PATH (`brew install ffmpeg`)
- Transcripts normalized: `npm run lyrics:normalize`
- OAuth credentials: `node scripts/shorts/oauth-init.js` (one-time setup)

## Workflow

```
1. Identify hooks     →  npm run shorts:identify
2. Review hook file      INGEST/shorts/pending/hooks-TIMESTAMP.json
3. Compose Short      →  npm run shorts:compose -- --hook INGEST/shorts/pending/hooks-TIMESTAMP.json --index 0 --audio /path/to/master.wav
4. Watch the MP4         Open INGEST/shorts/pending/short-[slug]-hook[N]-*.mp4
5. Approve              Move to INGEST/shorts/approved/ (manual)
6. Upload             →  npm run shorts:upload -- --file INGEST/shorts/approved/short-*.mp4
7. Confirm "yes"         Read the full metadata display, type yes
8. Publish              Open YouTube Studio link → set to Public when ready
```

## Hard Rules

- **Never auto-upload.** Every upload requires interactive "yes" confirmation.
- **Always uploads as PRIVATE.** Brandon manually promotes to Public in YouTube Studio.
- **Audio from Brandon's local master files only.** Never yt-dlp.
- **Never modify existing YouTube videos** — this pipeline is upload-only.

## Directory Layout

```
INGEST/shorts/
  pending/
    hooks-TIMESTAMP.json      ← output of shorts:identify
    short-[slug]-hook[N]-*.mp4  ← output of shorts:compose (awaiting review)
  approved/
    short-[slug]-hook[N]-*.mp4  ← manually moved here after watching
  uploaded/
    short-[slug]-hook[N]-*.mp4  ← moved here after successful upload
  shorts-log.json              ← upload log (slug, videoId, timestamp)
```

## File Naming

| File | Pattern |
|------|---------|
| Hook file | `hooks-2026-06-10T14-30-00.json` |
| Pending MP4 | `short-out-of-nowhere-hook0-2026-06-10T14-30-00.mp4` |
